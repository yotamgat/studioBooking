# 🔧 פתרון בעיה - היומן הישן עדיין מוצג

## הבעיה:
היומן שמוצג הוא עדיין הישן (עם שעות שלמות בלבד) במקום היומן החדש (עם רבעי שעה).

## הסיבה:
הדפדפן או Next.js עדיין משתמשים בגרסה ישנה מה-cache.

---

## ✅ הפתרון (5 שלבים):

### שלב 1: עצור את השרת
במסוף (Terminal), לחץ:
```
Ctrl + C
```

### שלב 2: מחק את תיקיית .next
```bash
rm -rf .next
```

או בחלונות:
```bash
rmdir /s .next
```

### שלב 3: הרץ את השרת מחדש
```bash
npm run dev
```

### שלב 4: נקה את cache הדפדפן
**Chrome/Edge:**
1. לחץ `Ctrl + Shift + Delete` (או `Cmd + Shift + Delete` במק)
2. בחר "Cached images and files"
3. לחץ "Clear data"

**או פשוט:**
- לחץ `Ctrl + Shift + R` (או `Cmd + Shift + R` במק) כדי לרענן בלי cache

### שלב 5: רענן את הדף
פתח מחדש את:
```
http://localhost:3000/booking
```

## 🔍 אם עדיין לא עובד:

### בדיקה 1: ודא שהקובץ הנכון קיים
```bash
ls -la components/booking/ImprovedCalendar.tsx
```

צריך להיות קובץ בגודל ~8-9KB.

### בדיקה 2: בדוק את הקונסול
1. פתח את הדפדפן
2. לחץ F12
3. לך ל-Console
4. יש שגיאות אדומות?

### בדיקה 3: בדוק איזה קומפוננטה נטענת
1. פתח F12 → Sources
2. חפש "ImprovedCalendar"
3. פתח את הקובץ
4. ודא שיש בו את הקוד החדש

### בדיקה 4: בדוק Network
1. F12 → Network
2. רענן את הדף
3. חפש קריאה ל-`available-slots`
4. יש כזו? מה התשובה?

## 🎯 תוצאה סופית:

אחרי כל השלבים, כשנכנסים ל-/booking ובוחרים חלל, אמור להיות:

1. **רשימת ימים** (לא טבלה)
2. כל יום מראה **"X משבצות פנויות"**
3. לחיצה על יום פותחת **רשימת משבצות ב-15 דקות:**
   - 09:00, 09:15, 09:30, 09:45
   - 10:00, 10:15, 10:30, 10:45
   - וכן הלאה...
4. לחיצה על משבצת → עובר לפרטי הזמנה עם השעה כבר מוגדרת
